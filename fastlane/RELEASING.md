# Releasing Off Grid mobile

Hand-written guide. Note: `fastlane/README.md` is auto-generated by fastlane (it gets
overwritten every time fastlane runs), so durable docs live here.

Off Grid ships no telemetry; fastlane's own anonymous analytics are disabled in the
Fastfile via `opt_out_usage`.

## Setup

```
bundle install                            # installs fastlane (see ../Gemfile)
cp fastlane/.env.example fastlane/.env     # fill in store credentials (gitignored)
```

### Local build prerequisites (gotchas)

Local Android builds need a specific toolchain. CI handles these automatically; locally
you must set them yourself or builds fail or hang:

- `ANDROID_HOME` must point at the SDK, e.g.
  `export ANDROID_HOME=$HOME/Library/Android/sdk` (or add `sdk.dir` to
  `android/local.properties`). Without it: "SDK location not found".
- **JDK 17 is required** (CI pins temurin 17). JDK 24 breaks the native `op-sqlite`
  CMake step ("A restricted method in java.lang.System has been called"). Set
  `JAVA_HOME` to a JDK 17 and stop any stale Gradle daemon on another JDK
  (`cd android && ./gradlew --stop`) - a mixed-JDK daemon can deadlock the build at
  0% CPU.

### Build-only verification (no upload, no secrets)

CI runs the lane on every push via the `fastlane-build` job in
`.github/workflows/ci.yml`: it runs `fastlane android build` on a clean JDK-17 runner and
uploads the resulting AAB as a build artifact. This is the recommended way to verify the
release pipeline without touching the stores or wrestling local toolchain setup.

Credentials are read from the environment (CI provides them as GitHub Actions secrets).
Nothing secret is committed: `fastlane/.env`, the Play key, the App Store Connect key,
and `.p8` files are gitignored.

## Lanes

```
bundle exec fastlane lanes               # list all lanes
bundle exec fastlane bump [type:patch]   # bump version in package.json + android + ios
bundle exec fastlane android beta        # build AAB -> Play internal track (draft)
bundle exec fastlane android release     # build AAB -> Play production (draft)
bundle exec fastlane android metadata    # push Android store listing only (no build)
bundle exec fastlane ios beta            # build IPA -> TestFlight
bundle exec fastlane ios release         # build IPA -> App Store (not submitted)
bundle exec fastlane ios metadata        # push App Store listing only (no build)
```

Release lanes upload as **draft / not-submitted** on purpose - a human confirms the
rollout in Play Console / App Store Connect. Store submission stays a deliberate step.

The `bump` lane does NOT commit or tag; the release GitHub Actions workflow handles the
commit, tag, and GitHub Release (see scripts/release.sh for the current manual flow this
mirrors).

### Required env per lane
| Lane | Needs |
|------|-------|
| `android *` | `PLAY_STORE_JSON_KEY_PATH` (Play service-account JSON) |
| `ios *` | `APP_STORE_CONNECT_API_KEY_PATH` (ASC API key) |
| `ios *` signing | `MATCH_GIT_URL` + `MATCH_PASSWORD` (optional; falls back to Xcode automatic signing) |

A lane fails fast with a clear message if its credentials are missing.

## Store metadata (Supply structure)

`fastlane/metadata/` follows the [Supply](https://docs.fastlane.tools/actions/supply/)
structure (also used by F-Droid / IzzyOnDroid). Edit the files, then run the `metadata`
lane.

```
fastlane/metadata/android/en-US/
├── title.txt                 Short app name (max 50 chars)
├── short_description.txt      One-liner (max 80 chars)
├── full_description.txt       Long description (~4000 chars)
├── changelogs/<versionCode>.txt   Release notes per version (max 500 chars)
└── images/
    ├── icon.png              512x512 app icon (TODO)
    ├── featureGraphic.png    1024x500 (TODO)
    └── phoneScreenshots/     4-8 screenshots, 320-3840 px (TODO)
```
